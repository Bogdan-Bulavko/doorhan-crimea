# Региональные поддомены

## Настройка виртуальных поддоменов для локальной разработки

Для работы с региональными поддоменами в локальной среде необходимо настроить DNS.

### Windows

1. Откройте файл hosts от имени администратора:
   - Путь: `C:\Windows\System32\drivers\etc\hosts`
   - Добавьте следующие строки:
   ```
   127.0.0.1 simferopol.localhost
   127.0.0.1 yalta.localhost
   127.0.0.1 alushta.localhost
   127.0.0.1 sevastopol.localhost
   ```

2. Перезапустите браузер

### macOS/Linux

1. Откройте терминал с правами суперпользователя
2. Редактируйте файл `/etc/hosts`:
   ```bash
   sudo nano /etc/hosts
   ```
3. Добавьте следующие строки:
   ```
   127.0.0.1 simferopol.localhost
   127.0.0.1 yalta.localhost
   127.0.0.1 alushta.localhost
   127.0.0.1 sevastopol.localhost
   ```

### Доступные регионы

- `localhost:3000` - основной сайт (default)
- `simferopol.localhost:3000` - региональный сайт Симферополя
- `yalta.localhost:3000` - региональный сайт Ялты
- `alushta.localhost:3000` - региональный сайт Алушты
- `sevastopol.localhost:3000` - региональный сайт Севастополя

### Как это работает

1. Middleware (`src/middleware.ts`) определяет регион по поддомену из заголовка `host`
2. Регион проксируется через заголовок `x-region` в запросах
3. Компоненты используют региональные метаданные из `src/app/metadata/regions.ts`
4. Контент адаптируется под выбранный регион

### Примечание

В современных браузерах `.localhost` автоматически резолвится в `127.0.0.1`, поэтому дополнительная настройка hosts может не потребоваться.
